YouTube POSTer - Deployment Guide

This guide covers deploying YouTube POSTer to various platforms.

Choosing a Platform

We recommend Render.com for its simplicity and free tier, but the application can be deployed to any Node.js hosting platform.

Platform Pros Cons
Render.com Free tier, Easy setup, Auto-deploy from Git Free tier spins down after inactivity
Heroku Popular, Many integrations No free tier anymore
Railway Developer-friendly, Good free tier Relatively new platform
Google Cloud Run Generous free tier, Scales to zero Requires Docker knowledge
AWS / DigitalOcean Full control, High performance More complex, Costs money

Deployment: Render.com (Recommended)

Step-by-Step Render.com Deployment

1. Prepare Your Repository

Ensure your code is pushed to GitHub, GitLab, or Bitbucket.

2. Create Render Account

  1. Go to render.com
  2. Sign up with GitHub (recommended for easy repo access)

3. Create a New Web Service

  1. Click "New +" → "Web Service"
  2. Connect your repository
  3. Configure:
    • Name: youtube-poster (or your choice)
    • Environment: Node
    • Region: Choose closest to your users
    • Branch: main (or your default branch)
    • Build Command: npm install
    • Start Command: npm start
    • Plan: Free

4. Add Environment Variables

In the Environment section, add:

GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
GOOGLE_REDIRECT_URI=https://your-app.onrender.com/oauth/callback
API_KEY=your-secure-api-key
NODE_ENV=production
PORT=3000
💡 Note: Render will provide you with a URL like https://youtube-poster.onrender.com. Update GOOGLE_REDIRECT_URI with this URL.

5. Update Google Cloud Console

  1. Go to Google Cloud Credentials
  2. Edit your OAuth 2.0 Client ID
  3. Add Authorized redirect URI: https://your-app.onrender.com/oauth/callback
  4. Save

6. Deploy

  1. Click "Create Web Service"
  2. Wait for the build and deployment to complete (5-10 minutes)
  3. Visit your app URL to verify it's running

7. Complete OAuth Setup

  1. Visit https://your-app.onrender.com/oauth/authorize
  2. Complete the Google OAuth flow
  3. Tokens will be saved on the server
✓ Deployment Complete! Your API is now live and ready to use.

Deployment: Railway

Railway Deployment

  1. Go to railway.app
  2. Sign up with GitHub
  3. Click "New Project" → "Deploy from GitHub repo"
  4. Select your repository
  5. Add environment variables (same as Render)
  6. Railway will auto-detect Node.js and deploy
  7. Get your deployment URL from the dashboard
  8. Update Google OAuth redirect URI
  9. Complete OAuth authorization

Deployment: Google Cloud Run

Google Cloud Run (Docker-based)

1. Create Dockerfile

FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 3000

CMD ["npm", "start"]

2. Build and Deploy

# Set your project ID
gcloud config set project YOUR_PROJECT_ID

# Build the container
gcloud builds submit --tag gcr.io/YOUR_PROJECT_ID/youtube-poster

# Deploy to Cloud Run
gcloud run deploy youtube-poster \
  --image gcr.io/YOUR_PROJECT_ID/youtube-poster \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated \
  --set-env-vars "GOOGLE_CLIENT_ID=xxx,GOOGLE_CLIENT_SECRET=xxx,API_KEY=xxx"

3. Update OAuth redirect URI with Cloud Run URL

Deployment: Heroku

Heroku Deployment

# Install Heroku CLI
# https://devcenter.heroku.com/articles/heroku-cli

# Login
heroku login

# Create app
heroku create youtube-poster

# Set environment variables
heroku config:set GOOGLE_CLIENT_ID=xxx
heroku config:set GOOGLE_CLIENT_SECRET=xxx
heroku config:set GOOGLE_REDIRECT_URI=https://youtube-poster.herokuapp.com/oauth/callback
heroku config:set API_KEY=xxx
heroku config:set NODE_ENV=production

# Deploy
git push heroku main

# Open app
heroku open

Production Considerations

1. Token Persistence

⚠️ Important: The default file-based token storage may not persist on platforms that use ephemeral file systems. Consider using:
  • Database: MongoDB, PostgreSQL, MySQL
  • Cloud Storage: AWS S3, Google Cloud Storage
  • Redis: For in-memory token storage

2. Environment Variables Security

  • Never commit .env files to version control
  • Use platform-specific secret management
  • Rotate API keys regularly
  • Use different keys for development and production

3. File Upload Limits

Default is 256 MB. For larger files:

  • Increase the limit in src/routes/upload.js
  • Configure your hosting platform's request size limits
  • Consider using resumable uploads for very large files

4. CORS Configuration

The app allows all origins by default. For production, consider restricting CORS:

// In src/app.js
app.use(cors({
  origin: ['https://your-frontend.com', 'https://another-domain.com']
}));

5. Rate Limiting

Add rate limiting to prevent abuse:

npm install express-rate-limit

// In src/app.js
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 10 // limit each IP to 10 requests per windowMs
});

app.use('/api/', limiter);

6. Logging and Monitoring

  • Use a logging service (e.g., Logtail, Papertrail)
  • Monitor uptime with UptimeRobot or similar
  • Track errors with Sentry or similar
  • Monitor API usage and quotas

7. HTTPS

Most modern platforms (Render, Heroku, Railway, Cloud Run) automatically provide HTTPS. Always use HTTPS in production to protect API keys and user data.

Scaling Considerations

Horizontal Scaling

If using multiple instances:

  • Use a shared database for token storage (not local files)
  • Consider a session store like Redis
  • Ensure OAuth tokens are accessible to all instances

Performance Optimization

  • Enable gzip compression
  • Use a CDN for static assets
  • Implement caching where appropriate
  • Optimize video processing pipeline

Troubleshooting

Deployment Failed

  • Check build logs for errors
  • Ensure all dependencies are in package.json
  • Verify Node.js version compatibility

OAuth Not Working

  • Verify redirect URI matches exactly in Google Console
  • Check environment variables are set correctly
  • Ensure HTTPS is being used (not HTTP)

Uploads Failing

  • Check file size limits
  • Verify OAuth token is valid
  • Check YouTube API quotas
  • Review server logs for errors

YouTube POSTer - Setup Guide

This guide will walk you through setting up the YouTube POSTer API from scratch.

Prerequisites

  • Node.js 14+ installed
  • A Google account
  • A YouTube channel associated with your Google account

Step 1: Google Cloud Console Setup

1.1 Create a Google Cloud Project

  1. Go to Google Cloud Console
  2. Click "Select a project" → "New Project"
  3. Enter a project name (e.g., "YouTube POSTer")
  4. Click "Create"

1.2 Enable YouTube Data API v3

  1. In your project, go to "APIs & Services" → "Library"
  2. Search for "YouTube Data API v3"
  3. Click on it and press "Enable"

1.3 Create OAuth 2.0 Credentials

  1. Go to "APIs & Services" → "Credentials"
  2. Click "Create Credentials" → "OAuth client ID"
  3. If prompted, configure the OAuth consent screen:
    • User Type: External (for testing) or Internal (for organization)
    • App name: YouTube POSTer
    • User support email: Your email
    • Developer contact: Your email
    • Save and Continue through all steps
  4. For Application type, select "Web application"
  5. Name: YouTube POSTer Web Client
  6. Authorized redirect URIs: Add your callback URL
    • For local: http://localhost:3000/oauth/callback
    • For production: https://your-domain.com/oauth/callback
  7. Click "Create"
  8. Save your Client ID and Client Secret
⚠️ Important: Never commit your Client Secret to version control. Keep it secure!

Step 2: Local Installation

2.1 Clone and Install

git clone <repository-url>
cd youtube-POSTer
npm install

2.2 Configure Environment Variables

Copy the example environment file:

cp .env.example .env

Edit .env and fill in your values:

# Server Configuration
PORT=3000
NODE_ENV=development

# Google OAuth Credentials
GOOGLE_CLIENT_ID=your-client-id-here.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret-here
GOOGLE_REDIRECT_URI=http://localhost:3000/oauth/callback

# API Key for client authentication
API_KEY=your-secure-random-api-key-here

# Token storage path
TOKEN_PATH=./tokens/youtube-token.json
💡 Tip: Generate a secure API key using: 
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Step 3: OAuth Authorization

3.1 Start the Server

npm start

The server should start on http://localhost:3000

3.2 Complete OAuth Flow

  1. Open your browser and go to: http://localhost:3000/oauth/authorize
  2. You'll be redirected to Google's OAuth consent screen
  3. Sign in with your Google account
  4. Review the permissions requested (YouTube upload access)
  5. Click "Allow"
  6. You'll be redirected back to the success page
✓ Success! Your OAuth tokens are now saved and the API is ready to use.

3.3 Verify Setup

Check the authentication status:

curl http://localhost:3000/oauth/status

Expected response:

{
  "authenticated": true,
  "message": "YouTube API is authorized and ready to use"
}

Step 4: Render.com Deployment

4.1 Prepare for Deployment

Make sure your package.json has a start script:

"scripts": {
  "start": "node src/server.js"
}

4.2 Create Render.com Service

  1. Go to Render.com and sign up/login
  2. Click "New +" → "Web Service"
  3. Connect your GitHub repository
  4. Configure the service:
    • Name: youtube-poster
    • Environment: Node
    • Build Command: npm install
    • Start Command: npm start
  5. Add environment variables:
    • GOOGLE_CLIENT_ID
    • GOOGLE_CLIENT_SECRET
    • GOOGLE_REDIRECT_URI (update with your Render URL)
    • API_KEY
    • NODE_ENV=production
  6. Click "Create Web Service"

4.3 Update Google Cloud Redirect URI

  1. Get your Render.com service URL (e.g., https://youtube-poster.onrender.com)
  2. Go back to Google Cloud Console → Credentials
  3. Edit your OAuth 2.0 Client ID
  4. Add the production redirect URI: https://your-service.onrender.com/oauth/callback
  5. Save

4.4 Complete OAuth on Production

  1. Visit https://your-service.onrender.com/oauth/authorize
  2. Complete the OAuth flow again for the production environment
⚠️ Note: Render.com free tier services may spin down after inactivity. The OAuth tokens will persist as long as you use a persistent storage solution (see Advanced Configuration).

Advanced Configuration

Persistent Token Storage

For production, consider using a database or cloud storage for tokens instead of the local file system. The token service can be extended to use:

  • MongoDB
  • PostgreSQL
  • Google Cloud Storage
  • AWS S3

Troubleshooting

OAuth Error: redirect_uri_mismatch

Make sure the redirect URI in your .env file exactly matches what's configured in Google Cloud Console.

Upload fails with authentication error

Your OAuth token may have expired. Visit /oauth/authorize again to re-authenticate.

File upload size limit

The default limit is 256 MB. For larger videos, adjust the limits in src/routes/upload.js.

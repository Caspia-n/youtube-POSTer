YouTube POSTer - API Usage Guide

This guide explains how to use the YouTube POSTer API to upload videos to YouTube.

Authentication

All API endpoints (except OAuth endpoints) require authentication using an API key. Include the API key in one of two ways:

Option 1: HTTP Header (Recommended)

X-API-Key: your-api-key-here

Option 2: Query Parameter

?apiKey=your-api-key-here
💡 Tip: Using the HTTP header is more secure as query parameters may be logged by proxies and servers.

Base URL

Local development: http://localhost:3000

Production: https://your-service.onrender.com

API Endpoints

POST /api/video

Upload a video to YouTube.

Request

Content-Type: multipart/form-data

Headers

Header Required Description
X-API-Key Yes Your API key for authentication

Form Data Parameters

Parameter Type Required Description
video File Yes The video file to upload (max 256 MB)
title String No Video title (default: "Untitled Video")
description String No Video description
tags JSON Array No Video tags (e.g., '["tag1","tag2"]')
privacyStatus String No public, unlisted, or private (default: "private")
categoryId String No YouTube category ID (default: "22" - People & Blogs)
madeForKids Boolean No "true" or "false" (default: "false")

Response (Success - 200 OK)

{
  "success": true,
  "message": "Video uploaded successfully",
  "video": {
    "id": "dQw4w9WgXcQ",
    "title": "My Awesome Video",
    "description": "This is a great video",
    "privacyStatus": "private",
    "url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ"
  }
}

Response (Error - 401 Unauthorized)

{
  "error": "Unauthorized",
  "message": "API key is required..."
}

Code Examples

cURL

curl -X POST http://localhost:3000/api/video \
  -H "X-API-Key: your-api-key-here" \
  -F "video=@/path/to/video.mp4" \
  -F "title=My Awesome Video" \
  -F "description=Check out this cool video!" \
  -F 'tags=["tutorial","demo"]' \
  -F "privacyStatus=private"

JavaScript (Node.js with axios)

const axios = require('axios');
const FormData = require('form-data');
const fs = require('fs');

async function uploadVideo() {
  const formData = new FormData();
  formData.append('video', fs.createReadStream('/path/to/video.mp4'));
  formData.append('title', 'My Awesome Video');
  formData.append('description', 'Check out this cool video!');
  formData.append('tags', JSON.stringify(['tutorial', 'demo']));
  formData.append('privacyStatus', 'private');

  try {
    const response = await axios.post('http://localhost:3000/api/video', formData, {
      headers: {
        'X-API-Key': 'your-api-key-here',
        ...formData.getHeaders()
      }
    });
    console.log('Upload successful:', response.data);
  } catch (error) {
    console.error('Upload failed:', error.response?.data || error.message);
  }
}

Python (with requests)

import requests

def upload_video():
    url = 'http://localhost:3000/api/video'
    headers = {
        'X-API-Key': 'your-api-key-here'
    }
    
    files = {
        'video': open('/path/to/video.mp4', 'rb')
    }
    
    data = {
        'title': 'My Awesome Video',
        'description': 'Check out this cool video!',
        'tags': '["tutorial","demo"]',
        'privacyStatus': 'private'
    }
    
    response = requests.post(url, headers=headers, files=files, data=data)
    
    if response.status_code == 200:
        print('Upload successful:', response.json())
    else:
        print('Upload failed:', response.json())

JavaScript (Browser/Frontend)

async function uploadVideo(file) {
  const formData = new FormData();
  formData.append('video', file);
  formData.append('title', 'My Awesome Video');
  formData.append('description', 'Check out this cool video!');
  formData.append('tags', JSON.stringify(['tutorial', 'demo']));
  formData.append('privacyStatus', 'private');

  try {
    const response = await fetch('http://localhost:3000/api/video', {
      method: 'POST',
      headers: {
        'X-API-Key': 'your-api-key-here'
      },
      body: formData
    });

    const result = await response.json();
    if (response.ok) {
      console.log('Upload successful:', result);
    } else {
      console.error('Upload failed:', result);
    }
  } catch (error) {
    console.error('Error:', error);
  }
}

// Example usage with file input
document.getElementById('fileInput').addEventListener('change', (e) => {
  const file = e.target.files[0];
  uploadVideo(file);
});

OAuth Management Endpoints

GET /oauth/status

Check if the API is authorized with YouTube.

Response

{
  "authenticated": true,
  "message": "YouTube API is authorized and ready to use"
}

GET /oauth/authorize

Start the OAuth flow to authorize the application. Redirects to Google's consent screen.

POST /oauth/revoke

Revoke the OAuth authorization (for testing/reset purposes).

Response

{
  "success": true,
  "message": "Authorization revoked successfully"
}

Error Handling

The API uses standard HTTP status codes:

Status Code Meaning
200 Success
400 Bad Request - Missing or invalid parameters
401 Unauthorized - Missing or invalid API key
403 Forbidden - Invalid API key
500 Internal Server Error - Something went wrong

Best Practices

  • Always use HTTPS in production to protect your API key
  • Keep your API key secret - don't expose it in client-side code
  • Handle errors gracefully - check response status codes
  • Use appropriate privacy settings - default is "private" for safety
  • Validate video files before uploading to avoid errors
  • Set meaningful titles and descriptions for better discoverability
⚠️ Rate Limits: YouTube Data API has quota limits. Be mindful of the number of uploads. See YouTube API quota documentation.
